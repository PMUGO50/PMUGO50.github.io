关联github远程仓库

发表于 2024-03-22 更新于 2024-03-23 分类于 工具使用
记录创建github仓库并与本地关联的全流程，主要是备忘用

记性差的结果就是需要每次用完一个工具后隔几天可能就忘了怎么使用，而且鉴于本来也不是计算机专业的，对 git 的理解也不是很深刻，所以只能通过记录的方式做个备忘

同时，这里也并不会叙述 git 的原理，以及它和其它版本控制系统的区别之类的，仅仅是记录基础的使用方法而已；并且远程仓库也不一定非要用 github ，任何采用 git 系统的仓库都可以

Windows 系统上关联远程仓库

Git 安装

直接在 git 的官网 https://git-scm.com/download/win 上按自己操作系统的位数选择相应的 git 下载即可，下载后点击进入安装，一路 ok 下来安装就行，注意安装目录不要有中文即可

点击页面右上角再打开右侧边栏，然后点击 “Your repositories” 进入仓库页面。点击右上角的 “New” 创建一个新仓库，进入如图界面

Repository name 即仓库名，按自己的想法设置即可；Description 是一句对该仓库功能或内容简单的描述，也按自己的想法写就好；Public 和 Private 是仓库是否向全网公开，根据需求选择即可；Add a README file 是在初始化的时候是否包含 README 文件，这个选项也是看自己需求，反正之后也能自己再写 readme ，这里就习惯性选上了

这里值得提的一点是正如上面的图所示，github 提供了 https 和 ssh 两种关联方式。采用 https 关联的话，在每次推送或者拉取时要通过账号密码进行登录，而 ssh 直接通过之前产生的密钥进行登录，更加安全也更加快捷

然后就是，windows 上选 https 的话，在操作上和 ssh 也没有什么区别，不会出现账号密码登录的提示；但在 linux 上就会提示登录账号密码，并且输入完后还会报错，理由是该方法已经在 2021 年 8 月被弃用了

总之就是，采用 ssh 进行关联就好

一些问题

最后有一个问题就是拉取的时候采用 git pull 指令会直接将本地的同名文件用远程仓库内的内容覆盖掉，刚才没有出现这个情况的原因是远程仓库中没有 main.py 和 amod.py ，所以要格外注意 commit, pull, push 的顺序问题，在 commit 提交之后再 pull 就算覆盖了本地文件也没有问题，因为更改已经提交过了，只要推送出去就可以

还有一点值得注意的是，git 系统单次提交的内容大小是有限的，好像是不能超过 50M 还是 100M，总之如果要提交的文件太大的话，要么分开提交，要么想其它办法
