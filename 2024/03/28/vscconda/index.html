在VSCode上配置Python运行环境

发表于 2024-03-28 更新于 2024-04-01 分类于 工具使用
叙述直接在 VSC 上配置或者用 Conda 配置 Python 运行环境的流程

有一段时间为了机器学习以及爬虫，需要有多个 python 环境，所以需要用 conda 创建虚拟环境，再让 VSC 调用虚拟环境来调试和运行 py 文件。突然想起来这件事情，所以临时起意回忆一下当时配置的过程

VSCode 下载与准备

VSCode 本身只是个文本编辑器，跟 Windows 自带的笔记本本质上是一样的东西。不过得益于繁多丰富的插件，才让其具备了许多编辑器本身并没有的功能，比如解释并运行代码、SSH 远程登录、追番等等

VSCode 是开源免费的，直接在官网 https://code.visualstudio.com 下载并安装就行

然后打开后点击右边栏中由四个方块构成的图标（下图中的自上向下数的第五个），这是浏览及管理插件的地方。在搜索栏里搜 chinese ，会弹出 chinese(simplified) ，即简体中文的插件，下面也有繁体的。看个人喜好简体还是繁体，总之点开之后安装，重启 VSC 界面就成为中文的了

接着安装 python 和 debug 的插件，在插件搜索栏中键入 python 后会出现一列插件，在其中找到 python 和 python debugger 两个插件安装，如下图左侧栏目中第一个和第三个，都是 Microsoft 自己发布的

安装完之后，再在搜索栏中键入 pylance 并安装，这个插件是为写 python 代码提供辅助功能的插件，包括函数的参数提示、引用跳转等。这个名字好像是来自骑士 Lancelot ，乐（Gawain 和 Tristan 流下了羡慕的泪水，Altria 继续干饭）

VSC 的配置到这里先暂停，为了调用环境，接下来需要下载和配置 Conda

Conda 创建虚拟环境

Conda 安装

Conda 本质上可以看成一个管理工具，就跟手机设置里能管理一堆 app 一样。Anaconda 集成了 Conda 本身和一些科学数据库，以及编辑运行 Python 的工具，对于专门进行 Python 开发的人员应该会很有帮助，官网下载地址是 https://www.anaconda.com/download

但是 Anaconda 占用的空间也很大，如果仅仅是为了配置环境可以考虑轻量版的 Miniconda ，Miniconda 相比于 Anaconda ，主要成分只有 Conda 本身，而没有集成一堆开发工具，甚至没有图形化用户界面。当然这不是问题，对于配置环境而言，常用的 conda 指令也就只有几条，命令行中输入也很快

这里就以 Miniconda 为例来创建环境，可供下载的版本在 https://repo.anaconda.com/miniconda 中列出，按自己的系统下载合适的版本，选择安装目录进行安装就可以，比如此处下载的是 23.11.0 ，安装在 D:/ 下

安装过程中可能会出现下面的选项

  • Add miniconda to my PATH environment variable，是说将 miniconda 自动添加到环境变量中，这可以通过手动完成，如果自动完成据说 conda 会加一些无关的环境变量，所以不勾选
  • Register miniconda as my default python，是说登记 conda 环境里的 python ，以便 VSC 等编辑器能找到，勾选上即可

然后就可以打开 cmd ，输入相应的 conda 命令来操作了

为了检查是否安装成功以及版本，打开 cmd 后直接输入

如果跳出版本号，则说明安装成功

VSCode 调用环境

对于已经创建并配置好的环境，可以用 VSC 进行调用

在 VSC 界面的左下角有设置的图标，打开并且点击设置，可以进入设置界面，接着如图在搜索栏中输入 python: default ，这里注意冒号是英文的，就会跳出图中的选项 Python: Default Intepreter Path ，即默认的解释器

将默认的解释器可以选为一个比较常用的环境下的 python.exe ，下图中 D:\Miniconda 是 Miniconda 安装的位置；envs 是 miniconda 安装根目录下的文件夹，专门用来装虚拟环境的；然后下面有创建好的各环境文件夹，比如这里是 flow ；打开 flow 之后会有解释器 python.exe ，将该解释器的绝对路径输入进去即可

配置好之后，在 VSC 中随便打开一个含 py 文件的文件夹，并且之后打开这个 py 文件，会发现右下角显示了当前所使用的虚拟环境名称，如下图中显示了 flow: conda

可以点击 flow: conda 这个地方，顶部就会出现可选的解释器（环境）列表，如下图

在这里可以选择要用的环境，如果采用默认环境自然不用变，如果要换环境的话点击相应的环境即可

然后要调试或运行 py 脚本的话，就在右边栏中选中如上图一样的“带有虫子（bug）的三角形”，并点击“创建 launch.json 文件”，点击后顶部会先后出现对调试器的选取以及要调试的文件的选取，分别选取一开始安装的“ python debugger ”以及“ python 文件”即可，选完后会弹出 launch.json 的内容

如果只是要运行当前脚本的话，就不用动这个文件，回到 py 脚本按 F5 即为调试，按 ctrl+F5 为直接运行，F10 为按过程运行，F11 为按语句运行

但如果比如说这是个小项目，有多个脚本，但是希望每次按 F5 都是从主程序进入来执行的话，那么就把 json 中的 “program” 项改为主程序的名字，比如一般是 main.py ，这样每次按 F5 或者 ctrl+F5 都会从主程序进入项目开始运行
