在FPGA上实现UART通信

发表于 2024-03-08 更新于 2024-03-22 分类于 数电/FPGA笔记
让 PC 机和 FPGA 以 UART 的协议进行通信，效果是从 PC 的键盘输入字符，FPGA 接收后立马再返回给 PC

UART 协议

UART 是应用于串口的一种协议，它约定一个数据流由 10 位或 11 位构成，分别是

  • 起始位 1 位，起始位为 0 时表示起始有效
  • 数据位 7 位或 8 位，接下来采用 8 位输入输出 ASCII 码
  • 奇偶校验位 1 位，接下来采用偶校验
  • 终止位 1 位

波特率是指每秒钟传送多少个码元，UART 典型的波特率可以设置为 9600 。一个码元可能含有多位数据，当一个码元只含有一个数据时，此时波特率等于比特率，也就是 9600 波特率就意味着每秒传输 9600 位数据，等于 9600 比特率

控制模块

控制模块是几个模块中相对而言最为复杂的，它取决于具体要实现的功能。这里希望实现的功能是：以键盘作为输入，显示屏为输出，在数据输入之前显示

然后在冒号后每当输入一个字符，就显示一个字符，直到输入一个换行符（ LF ）为止。当检测到换行符后结束本行，并在下一行重新显示

重复以上过程。并且如果什么都不输入，那么就保持原样

对于这个功能，控制模块大致的工作行为是

  • 如果接收模块送来接收完成的通知，就向发送模块给出一个发送命令信号，将接收模块收到的数据发送出去；在此期间控制模块进入待机状态

  • 进入待机状态时，检测接收的数据是否是换行符 LF

    • 如果不是，那么维持待机状态，且如若又有接收完成的通知，那么就重新执行上一步
    • 如果是，那么跳出待机状态，进入提示状态

  • 进入提示状态后，依次向发送模块送出 ‘(‘, ‘e’, ‘v’, …, ‘t’, ‘:’, ‘ ‘，即字符串 “(even/odd parity)Please input: “ ，然后进入等待输入状态

  • 进入等待输入状态后，如果没有其它变化，就一直保持在该状态

因此，用一个有限状态机来表达的话，上述过程需要三个状态：待机、提示、等待输入。实际上在真正实现的过程中发现，在进入提示状态前需要有一个准备状态用来初始化一些参数，以供提示状态输出字符串时使用。所以最后需要四个状态：

  1. 待机状态：2’b00
  2. 提示准备状态：2’b01
  3. 提示状态：2’b10
  4. 等待状态：2’b11

这里面相对而言机能比较复杂的又是提示状态，因为要依次给发送模块送去字符，这部分的实现方式则是

  • 向发送模块给出一个发送命令信号，并将要发送的字符交给发送模块，等待该字符发送完毕
  • 重复上述过程，直到最后一个字符发送完毕，进入等待输入状态

接收模块

在代码中

  • 两个输入分别是 UART 的时钟 clk_uart ；以及从串口 RXD 输入的一位数据 bit_in

  • 三个输出分别是要送给控制模块的完整八位数据 din ；占线信号 idle，该信号用来防止同时接收和发送；接收完成信号 recfinish

  • reg 型变量中

    • bitin_pre 是上一时刻的 bit_in ，用来检测输入信号从 1 到 0 的下降沿
    • in_true 表示下降沿被检测到，初步判断准备进行接收
    • allowin 是 in_true 为真且 idle 空闲时开启，表示最终允许接收
    • presult 是奇偶校验中的
    • dataerror 是表示奇偶校验不通过

  • 参数 parity 是奇偶校验方式，这里采用偶校验

  • 第一个 always 块是检测 bit_in 的下降沿

  • 第二个 always 块是判断最终是否允许接收

  • 第三个 always 块是接收过程

    • 如果 allowin 为真，则将 idle 置为 1 表示占线，同时 counter 开始计数，每隔 16 取一次值，因为每位数据用 16 个时钟脉冲采样，第一位数据在 24 取的原因是第一位数据是 16~32 ，24 在中间，而靠近边缘数据可能不稳定

      并且在每次取数据的时候也计算出 presult ，即

    • 在 136 处取到最后一个数据位后，在 152 处取奇偶校验位，并与 presult 比较，如果一致则 dataerror 为 0 表示校验通过，反之为 1

    • 最后在 168 处取到终止位后，如果确为 1 而且 dataerror 为 0 ，说明接收成功，输出 recfinish 为 1 ，反之为 0 ；同时将占线信号置零，因为接收已经结束

    • 而如果 allowin 为假，那么将各变量初始化等待输入的来临

控制模块

在代码中

  • 三个输入分别是 UART 的时钟 clk_uart ；接收完成信号 recfinish ；接收到的 8 位数据 din

  • 两个输出分别是发送命令信号 outstart ；要发送的 8 位数据 dout

  • wire 型变量 string 存储了提示状态下要输出的信息，这里是

  • reg 型变量 k 是表示在提示状态下目前已输出了多少个字符

  • always 块中

    • 如果 recfinish 为真，就将 din 赋给 dout ，并将 outstart 从 0 转为 1 以激活发送模块，同时接下来进入待机状态 2’b00
    • 如果 recfinish 为假，则判断控制模块的所处状态
      1. 待机状态：将 outstart 重新置零，为下一个上升沿做准备；并判断 din 是否为换行符 LF（ ascii : 10 ），是则进入提示准备状态 2’b01，否则保持待机
      2. 提示准备状态：将 k 置零，outstart 也置零，counter 同样置零，然后进入提示状态 2’b10
      3. 提示状态：当 k 不满 28 时，每当 counter 归零就发送字符 string[k] ，然后等待 254 个时钟周期保证发送完成，对 k 加 1 ，重复该过程；而当 k 为 28 时，在 counter 为零时发送字符 string[28]，同样等待 254 个时钟周期保证发送完成，并进入等待输入状态 2’b11
      4. 等待输入状态：当无事发生时 FSM 停在该状态

发送模块

在代码中

  • 三个输入分别是 UART 的时钟 clk_uart ；从控制模块那里得来的 8 位完整输出 dout ；以及控制模块给出的发送开始命令 outstart

  • 两个输出分别是从串口 TXD 输出的一位数据 bit_out ；以及占线信号 idle

  • reg 型变量中

    • outstart_pre 是上一时刻的 outstart ，用来检测发送命令信号从 0 到 1 的上升沿
    • out_true 表示上升沿被检测到，初步判断准备进行发送
    • allowout 是 out_true 为真且 idle 空闲时开启，表示最终允许发送
    • presult 是奇偶校验中的

  • 参数 parity 是奇偶校验方式，这里采用偶校验

  • 第一个 always 块是检测 outstart 的下降沿

  • 第二个 always 块是判断最终是否允许接收

  • 第三个 always 块是发送过程

    • 如果 allowout 为真，则将 idle 置为 1 表示占线，同时 counter 开始计数，每隔 16 发送一位，注意第一位是起始位 0

      并且在每次取数据的时候也计算出 presult ，即

    • 在 128 处发送完最后一个数据位后，在 144 处发送奇偶校验位

    • 最后在 160 处发送终止位，在 168 处结束发送过程，将占线信号置零

与 PC 机的通信

在 PC 上安装驱动 CP210x_VCP_Windows ，可以在 https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers?tab=downloads 下载

然后用 USB 连接 AX309 和 PC 机，打开串口调试软件，这里用 teraterm ，在相关设置中设置串口参数，teraterm 是在 setup 的 serial port 中设置

参数 值
比特率 9600
数据位 8 bit
校验位 even
停止位 1 bit

同时，因为输出的换行（控制模块 line 14 ）采用了 LF 的方式

输入的换行（控制模块 line 54 ）也是 LF 的方式

所以在相关设置中还要设置接收和发送的新行都是 LF ，teraterm 是在 setup 的 terminal 中设置

然后打开 teraterm 就可以用键盘进行输入，输入会到 FPGA 中，然后再返回来在 teraterm 的命令行里输出。正如所设想的，每当输入换行符时，开启新行并等待新的输入
